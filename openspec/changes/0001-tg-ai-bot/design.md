# Design: Telegram AI-бот с Alice AI

## Current behavior

Сейчас в репозитории есть только примеры запросов к Alice AI и пустой `README.md`. Готового приложения Telegram-бота, хранилища состояния и конфигурационного слоя нет.

## Target behavior

Бот должен:

- принимать сообщения из Telegram через `aiogram`;
- проверять пользователя по allowlist;
- загружать роли из `prompts.json`;
- хранить по каждому пользователю:
  - активную роль;
  - историю последних сообщений;
- отправлять текст в Alice AI с системным промптом выбранной роли;
- поддерживать команду сброса;
- поддерживать отдельную команду генерации изображения;
- сохранять состояние в локальный JSON-файл.

## Affected modules/files

- `tg_ai_bot/main.py` - точка входа;
- `tg_ai_bot/bot.py` - сборка Telegram-роутеров и обработчиков;
- `tg_ai_bot/config.py` - загрузка `.env` и валидация настроек;
- `tg_ai_bot/prompts.py` - загрузка и проверка `prompts.json`;
- `tg_ai_bot/state.py` - JSON-хранилище пользовательского состояния;
- `tg_ai_bot/llm.py` - клиент Alice AI для текста и изображений;
- `prompts.json` - список ролей;
- `.env.example` - пример конфигурации;
- `requirements.txt` - зависимости;
- `tests/` - проверки ключевой логики;
- `README.md` - инструкция по запуску;
- `docs/changelog.md` - запись о первом релизе.

## Data model changes

Локальное состояние пользователя хранится в JSON в структуре:

```json
{
  "123456789": {
    "role": "assistant",
    "messages": [
      { "role": "user", "content": "Привет" },
      { "role": "assistant", "content": "Здравствуйте!" }
    ]
  }
}
```

Правила:

- ключ верхнего уровня - Telegram user id в виде строки;
- `role` - ключ роли из `prompts.json`;
- `messages` - список последних сообщений, обрезаемый по лимиту из `.env`.

## API changes

Внешнего HTTP API нет. Изменяется только Telegram-интерфейс бота.

## Config changes

Новые переменные окружения:

- `TELEGRAM_BOT_TOKEN`;
- `YANDEX_CLOUD_FOLDER`;
- `YANDEX_CLOUD_API_KEY`;
- `YANDEX_CLOUD_MODEL`;
- `YANDEX_CLOUD_MODEL_ART`;
- `MAX_CONTEXT_MESSAGES`;
- `TEMPERATURE`;
- `MAX_OUTPUT_TOKENS`;
- `ALLOWED_USER_IDS`;
- `STATE_FILE`;
- `PROMPTS_FILE`.

## DB/migration notes

База данных не используется. Состояние хранится в JSON-файле, который создается автоматически при первом сохранении.

## Error handling

- при отсутствии обязательных переменных окружения запуск завершается с понятной ошибкой;
- при неизвестной роли бот сообщает список доступных ролей;
- при недоступности Alice AI бот сообщает об ошибке без падения всего процесса;
- при поврежденном JSON-хранилище бот должен либо показать явную ошибку, либо создать новый файл после ручного восстановления;
- при недоступном пользователе бот отвечает отказом в доступе.

## Security implications

- allowlist снижает риск несанкционированного использования;
- API-ключи не логируются и не печатаются в чат;
- JSON-хранилище не должно содержать секреты;
- пользовательский ввод передается в LLM как данные, поэтому нельзя использовать его для построения кода или shell-команд.
