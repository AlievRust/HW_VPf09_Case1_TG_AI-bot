# Proposal: Telegram AI-бот с ролями и генерацией изображений

## Problem

В репозитории есть только примеры обращения к Alice AI, но нет готового Telegram-бота. Нужен приватный бот, который умеет:

- вести диалог с контекстом;
- переключать системные роли;
- сбрасывать контекст;
- генерировать изображения по отдельной команде.

## Goal

Сделать рабочий Telegram-бот на Python с `aiogram`, который использует Alice AI как текстовую и image-модель, хранит состояние по пользователю в JSON и настраивается через `.env`.

## Scope

- запуск Telegram-бота;
- текстовые ответы через Alice AI;
- хранение последних сообщений по пользователю;
- набор ролей в `prompts.json`;
- команды для просмотра ролей, смены роли и сброса контекста;
- команда генерации изображения;
- конфигурация через `.env`;
- документация по запуску.

## Out of scope

- веб-интерфейс;
- база данных и миграции;
- многопроцессная синхронизация состояния;
- админ-панель;
- логика платежей, подписок и лимитов;
- автоматическое обучение на пользовательских данных.

## User-visible behavior

- `/start` показывает краткую справку и текущую роль;
- `/roles` показывает доступные роли из `prompts.json`;
- `/role <key>` переключает активную роль;
- обычный текст отправляется в LLM с учетом истории диалога;
- `/reset` очищает историю и возвращает роль по умолчанию;
- `/img <prompt>` отправляет запрос на генерацию изображения и возвращает картинку в чат;
- бот отвечает только разрешенным пользователям;
- контекст хранится между перезапусками.

## Risks

- неправильная настройка `.env` может привести к неработающему запуску;
- JSON-хранилище требует аккуратной записи, чтобы не повредить состояние;
- форматы ответов Alice AI могут меняться и потребовать адаптации клиента;
- allowlist защищает доступ, но требует явного заполнения.

## Acceptance criteria

- бот запускается локально с конфигурацией из `.env`;
- текстовые сообщения используют историю последних сообщений по каждому пользователю;
- роли берутся из `prompts.json` и могут переключаться командой;
- `reset` очищает историю и возвращает роль по умолчанию;
- `img` возвращает изображение в чат;
- документация описывает запуск и конфигурацию;
- добавлены тесты для ключевой логики хранения и конфигурации.
